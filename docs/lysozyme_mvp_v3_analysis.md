## Main Results

### Solubility

Source benchmark mean:
- `source_sol_mean = 0.264`

Random task:
- `no_steering`: `sol_mean 0.645`, `delta +0.380`, `success_rate 1.00`, `percent_identity_mean 0.444`
- `naive_steering`: `sol_mean 0.477`, `delta +0.212`, `success_rate 0.833`, `percent_identity_mean 0.424`
- `alignment_steering`: `sol_mean 0.429`, `delta +0.165`, `success_rate 0.833`, `percent_identity_mean 0.425`

ASPO task:
- `naive_steering`: `sol_mean 0.618`, `delta +0.353`, `success_rate 0.833`, `percent_identity_mean 0.339`
- `alignment_steering`: `sol_mean 0.770`, `delta +0.505`, `success_rate 0.833`, `percent_identity_mean 0.362`

Interpretation:
- The low-solubility tail is now meaningfully optimizable.
- Under the `random` task, the plain no-steering baseline is already strong.
- Under the `aspo` task, `alignment_steering` is the best solubility method overall by a clear margin.
- Steering helps more when paired with ASPO than when paired with random masking.

### Thermostability

Source benchmark mean:
- `source_therm_mean = 46.51`

Random task:
- `no_steering`: `therm_mean 46.01`, `delta -0.50`, `success_rate 0.333`, `percent_identity_mean 0.408`
- `naive_steering`: `therm_mean 48.45`, `delta +1.94`, `success_rate 0.500`, `percent_identity_mean 0.349`
- `alignment_steering`: `therm_mean 46.42`, `delta -0.09`, `success_rate 0.500`, `percent_identity_mean 0.311`

ASPO task:
- `naive_steering`: `therm_mean 47.12`, `delta +0.61`, `success_rate 0.500`, `percent_identity_mean 0.279`
- `alignment_steering`: `therm_mean 48.25`, `delta +1.74`, `success_rate 0.667`, `percent_identity_mean 0.298`

Interpretation:
- The no-steering random baseline does not improve thermostability.
- Thermostability gains require steering.
- `random + naive_steering` gives the highest mean therm gain.
- `aspo + alignment_steering` gives the best success rate and a strong positive median gain, so it looks more consistent across sequences.

## Cross-Task Takeaways

- Solubility and thermostability prefer different edit regimes.
- For solubility, ASPO site selection is clearly useful. `alignment_steering` with ASPO is the strongest result in this set.
- For thermostability, both `random + naive_steering` and `aspo + alignment_steering` are competitive. The former has the highest mean gain; the latter looks more robust by success rate and median delta.

## Important Caveat About Edit Size

Even after allowing the original token again, the current pipeline still uses:
- `mask_ratio = 0.1`
- `num_rounds = 10`
- a shrinking candidate pool

That means every site is still selected exactly once across the 10 rounds. This is why:
- `edited_site_count` is still essentially the full sequence length
- percent identity improves relative to the old version, but the method is still closer to full-sequence refinement than sparse local editing

What changed is not the number of visited sites, but whether the model is allowed to keep the current residue at those sites.

This distinction matters:
- `edited_site_count` measures visited positions
- `hamming_distance` measures positions that truly changed

In the revised runs, `hamming_distance` is much lower than `edited_site_count`, which confirms that allowing the original token reduced destructive rewriting.

## Diversity Metric Caveat

`pairwise_hamming_mean` is `0.0` in these runs, but that should not be interpreted as the edited sequences being identical.

Reason:
- the diversity code only compares pairs with equal sequence length
- these lysozyme test subsets contain multiple distinct lengths
- with only `6` sequences per benchmark tail, there are effectively no equal-length cross-sequence pairs to average

So for this dataset, `pairwise_hamming_mean` is currently not informative.

## Recommended Next Steps

1. If the goal is sparse editing rather than whole-sequence refinement, decouple `num_rounds` from full coverage.
   For example: keep `mask_ratio=0.1`, but use `num_rounds=2` or allow site revisits instead of exhausting all positions.

2. Add a second edit-size regime to experiments.
   Suggested comparison:
   - low-edit: `mask_ratio=0.05`, `num_rounds=2`
   - high-edit: current `mask_ratio=0.1`, `num_rounds=10`

3. For lysozyme specifically, prioritize:
   - solubility: `aspo + alignment_steering`
   - thermostability: compare `random + naive_steering` against `aspo + alignment_steering`
